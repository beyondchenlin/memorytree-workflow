## North Star
Replace metadata-only heartbeat discovery cache reuse with full-content hash validation so unchanged transcript skipping stays correct even when file size and modified time are misleading.

## Scope
- Add per-source content-hash fingerprints to the heartbeat discovery cache.
- Use content hashes as the correctness gate for cache reuse and unchanged-source skipping.
- Keep the current transcript import statuses (`created`, `updated`, `unchanged`) and heartbeat scheduling behavior intact.
- Handle older cache documents safely when they do not contain the new hash field.
- Add regression coverage for content changes that keep the same file size and modified time.

## Acceptance Signals
- Discovery cache entries persist a full-content hash for each discovered source.
- Heartbeat does not treat a changed transcript as unchanged only because `size` and `mtime` still match.
- Existing cache files without content hashes are ignored or refreshed safely without crashes.
- Typecheck, test, and build all pass after the change.
