### N7 - MEDIUM: `parseIsoTimestamp` may silently misinterpret partial timestamps
`/src/transcript/common.ts` lines 84-96: The function appends `+00:00` to any string that does not match the timezone offset regex pattern. For strings like `"2024-01"` (year-month only) or `"invalid-text"`, it produces `"2024-01+00:00"` or `"invalid-text+00:00"`, then passes to `new Date()`. JavaScript's `Date` constructor is notoriously lenient with some partial formats on certain engines, potentially returning a valid but incorrect date rather than `NaN`. The `isNaN` guard protects against fully invalid strings, but edge cases (like `"2024-01+00:00"` being interpreted as January 2024) could silently succeed. The Python code used `dateutil.parser.parse` which has different leniency characteristics.

**File**: `/src/transcript/common.ts` lines 84-96
