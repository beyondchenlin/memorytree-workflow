### 10. Missing Features

- **CRITICAL -- No consecutive failure tracking.** The spec requires tracking consecutive failures across heartbeat runs (3 failures before alerting). No persistent state for this exists anywhere in the codebase. This would require storing failure counts per project in a state file (e.g., `~/.memorytree/state.json` or within alerts.json).

- **HIGH -- Config and lock modules don't log warnings.** `_config_utils.py` and `_lock_utils.py` are pure utility modules with no logger. Multiple spec requirements for logging warnings on config fallback and stale lock cleanup are unmet. The logging only happens at the `heartbeat.py` level, which doesn't cover cases handled silently inside these modules.

- **MEDIUM -- `status` subcommand does not show "last execution result."** The spec says it should show "last execution result" in addition to registration status. The code only shows registration, lock, and config. No last-run timestamp or exit code is displayed. Showing the last line of the heartbeat log or a stored last-run result would satisfy this.

- **MEDIUM -- `_python_path()` returns a string with spaces for `uv run python`.** On line 342 of `memorytree_daemon.py`, `_python_path()` returns `f"{uv} run python"` as a single string, but it's used in contexts where it's passed as the first element of a subprocess list (e.g., in cron line, schtasks `/tr`, launchd `ProgramArguments`). For the launchd plist, this puts `"uv run python"` as a single `<string>` element in `ProgramArguments`, which would fail because launchd expects each argument as a separate `<string>`. Similarly, in `cmd_run_once` (line 119), `subprocess.run([python, str(heartbeat_script)])` would try to find an executable named `"uv run python"` (with spaces) which would fail.

- **LOW -- `recall-session.py` uses `discover_source_files()` without project-scoped narrowing at the OS level.** The spec says "Scan all three client transcript directories for the current project." The code scans all directories and filters after parsing, which is correct but potentially slow for many transcripts.
