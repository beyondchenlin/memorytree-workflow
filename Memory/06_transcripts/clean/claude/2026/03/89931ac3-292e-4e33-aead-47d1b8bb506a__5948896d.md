## PR #18 — fix(daemon): suppress console window on Windows

**评价：整体质量不错，方案正确。**

**优点：**
- VBScript + `wscript.exe /B` 方案是 Windows 静默启动的标准做法
- UTF-16 LE BOM 编码考虑了中文用户名路径，细节到位
- install 失败时清理 `.vbs`，uninstall 对称删除，生命周期管理干净
- 注释清晰，解释了 `SW_HIDE(0)` 和 `False` 的含义

**问题：**

1. **[MEDIUM] `node` 路径硬编码** — VBScript 里直接写 `node`，依赖 PATH 环境变量。Task Scheduler 运行时 PATH 可能和用户 shell 不一致，如果用户用 nvm/fnm 管理 Node，`node` 可能找不到。考虑用 `process.execPath` 写入完整的 node 路径：
   ```ts
   // 现在
   `WshShell.Run "node ""${escaped}"" daemon run-once", 0, False`
   // 建议
   const nodePath = process.execPath.replace(/"/g, '""')
   `WshShell.Run """${nodePath}"" ""${escaped}"" daemon run-once", 0, False`
   ```

2. **[LOW] 没有新增测试** — `writeVbsLauncher` 是纯函数（写文件 + 返回路径），可以加单元测试验证生成的 VBS 内容和编码是否正确。现有 `test_daemon.py` 已有 21 个测试覆盖 install/uninstall，TS 侧也应该补上。
