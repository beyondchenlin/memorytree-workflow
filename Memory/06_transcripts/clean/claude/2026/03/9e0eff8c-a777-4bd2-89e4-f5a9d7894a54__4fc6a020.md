## Official Claude Code Skill Format and Structure

Based on official Anthropic documentation and your local skill implementation, here's the complete specification:

### 1. Required Directory Structure

```
skill-name/
├── SKILL.md (REQUIRED)
├── scripts/ (optional)
├── references/ (optional)
├── assets/ (optional)
└── agents/ (optional)
```

### 2. Required Files

**SKILL.md** is the ONLY required file. It contains:
- YAML frontmatter with metadata (at the top, between `---` markers)
- Markdown instructions that Claude follows when the skill is invoked

### 3. SKILL.md Format Specification

**Structure:**
```markdown
---
name: skill-name-in-lowercase
description: Clear, specific description (200 characters max) of when Claude should use this skill
---

# Skill Title

Detailed markdown instructions for Claude to follow...
```

**Required Frontmatter Fields:**
- `name`: Lowercase letters, numbers, hyphens only. Becomes the `/slash-command`
- `description`: Critical metadata. Claude uses this to decide when to automatically load your skill. Must be specific enough for Claude to select the right skill from 100+ possibilities (200 characters max)

**Optional Frontmatter Fields:**
- Version information
- Allowed tools/permissions
- Other configuration metadata

### 4. Supporting Directory Purposes

**`scripts/`** (optional)
- Executable code for deterministic/repetitive tasks
- Python scripts, shell scripts, or other runnable code
- Claude executes these for complex multi-step operations, data transformations, API interactions
- Referenced from SKILL.md when needed

**`references/`** (optional)
- Documentation and knowledge bases
- Detailed guides, API documentation, configuration specs
- Claude loads these into context as needed when SKILL.md references them
- Keep implementation details here to avoid cluttering the main instructions

**`assets/`** (optional)
- Files used in output: templates, icons, fonts, example outputs
- Subdirectories like `templates/` for template files
- Language variants like `assets/templates/en/`, `assets/templates/zh-cn/`

**`agents/`** (optional)
- Configuration files for sub-agents (e.g., `openai.yaml`)
- Used for multi-agent skill orchestration

### 5. Skill Installation and Discovery

**Installation Locations:**

| Location | Scope | Auto-discovered |
|----------|-------|-----------------|
| `~/.claude/skills/` | Global (all projects) | Yes |
| `.claude/skills/` | Project-specific | Yes |
| Nested `.claude/skills/` | Subdirectory-scoped | Yes (v2.1.6+) |

**Discovery Mechanism:**
- Claude automatically discovers skills in `.claude/skills/` directories
- Live change detection allows editing skills during sessions without restarting
- Nested skills discovery finds skills in subdirectory `.claude/skills/` when working with files in those subdirectories

### 6. Best Practices from Official Guidelines

**Description Quality:**
- The description is critical—Claude uses it to choose the right skill from potentially 100+ available
- Provide enough detail so Claude knows when to select this skill
- Rest of SKILL.md provides implementation details

**File Organization:**
- Keep SKILL.md focused on high-level instructions
- Move complex details to `references/` files
- Reference bundled resources from SKILL.md so Claude knows what they contain and when to load them

**Supporting Resources:**
- Only include resources that Claude will actually need
- Use references for documentation, scripts for deterministic operations
- Example: Your memorytree-workflow skill uses 10+ reference files for configuration details

### 8. How Skills Are Used by Claude

1. Claude reads the `description` field to decide whether to use your skill
2. When invoked (via `/slash-command` or automatically), Claude loads the full SKILL.md
3. Claude reads referenced `references/` files as needed for detailed context
4. Claude executes `scripts/` when deterministic operations are required
5. Claude fills in or uses `assets/` templates for output

---

**Key Takeaway:** SKILL.md is the gateway. Its frontmatter determines discovery, and its instructions determine execution. Everything else is supporting detail that gets loaded on demand.
