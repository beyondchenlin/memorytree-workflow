## TASK A: P1 Remaining -- CI Pipeline + Integration Tests

### A1: GitHub Actions CI Pipeline

**File to create:** `.github/workflows/ci.yml`

**Design decisions:**

1. **Runner strategy**: Use `pip install coverage` rather than `uv`. The project has zero external dependencies, so `uv` adds unnecessary complexity in CI. Plain `python` with `coverage` from PyPI is the simplest approach. `coverage` is the only CI dependency.

2. **Matrix**: Python 3.11, 3.12, 3.13 across ubuntu-latest, macos-latest, windows-latest (9 combinations).

3. **Test discovery**: The existing tests use `sys.path.insert(0, str(SCRIPTS_DIR))` relative to `__file__`, so they work from any working directory. Use `coverage run -m unittest discover -s tests -p "test_*.py"` from repo root. This will work because each test file resolves its own import paths.

4. **Coverage reporting**: Run `coverage report --fail-under=80` to enforce the 80% threshold. Also upload coverage as an artifact or use a comment on PRs.

5. **Triggers**: `push` to `master`, `pull_request` to `master`.

**Proposed workflow structure:**

```yaml
name: CI
on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        python-version: ["3.11", "3.12", "3.13"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - run: pip install coverage
      - run: coverage run -m unittest discover -s tests -p "test_*.py"
      - run: coverage report --fail-under=80
      - run: coverage xml
      - uses: actions/upload-artifact@v4
        if: matrix.os == 'ubuntu-latest' && matrix.python-version == '3.13'
        with:
          name: coverage-report
          path: coverage.xml
```

**Potential issues:**

- Windows path separator differences: The test files use `Path` throughout, so this should be fine. The `SCRIPTS_DIR = Path(__file__).resolve().parents[1] / "scripts"` pattern works on all platforms.
- The `test_upgrade_checks.py` uses `importlib.util.spec_from_file_location` for the hyphenated `upgrade-memorytree.py`. This works on all platforms.
- Some tests in `test_transcript_utils.py` create temp files with paths containing forward slashes in assertions. These should be fine since they use `Path.as_posix()`.

**Coverage concern:** The `coverage` module only tracks coverage for the test process. Since tests import from `scripts/`, coverage will track those modules. However, the entry-point scripts (`heartbeat.py`, `memorytree_daemon.py`, etc.) are not unit-tested -- their `main()` functions are not called in tests. The 80% threshold should still be achievable since the bulk of logic lives in the internal modules.
