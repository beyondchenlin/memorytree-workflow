- 还有测试问题。现在的 `init/upgrade` 单测直接调 `cmdInit/cmdUpgrade`，并没有隔离 HOME，见 [init-upgrade.test.ts](d:/demo1/memorytree-workflow/tests/cli/init-upgrade.test.ts#L20) 和 [init-upgrade.test.ts](d:/demo1/memorytree-workflow/tests/cli/init-upgrade.test.ts#L41)。`#25` 合进去后，这组测试会开始真实碰 `~/.memorytree/config.toml`，但 PR 本身没补对应测试或隔离。
- 它还会模糊掉一些当前“先 quick-start 再用”的引导，比如 [cmd-caddy.ts](d:/demo1/memorytree-workflow/src/cli/cmd-caddy.ts#L16) 现在明确要求项目先在 config 里注册。
