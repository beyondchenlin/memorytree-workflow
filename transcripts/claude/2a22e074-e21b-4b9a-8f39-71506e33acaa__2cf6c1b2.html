## 和 memorytree-workflow skill 冲突吗？

安装不会冲突。 它们是完全不同的东西：

| | CLI (@openmnemo/cli) | Skill (memorytree-workflow) |
|---|---|---|
| 安装位置 | npm 全局 (node_modules) | ~/.codex/skills/ |
| 使用者 | 人类用户在终端里用 | AI agent（Codex/Claude）自动调用 |
| 调用方式 | openmnemo init | agent 内部触发 |

但功能上有重叠 — 两者都能 init、import、discover 等。可能的情况：

- skill 内部可能调用 CLI 命令（那就是互补关系，没问题）
- skill 有自己独立的实现（那就是两套逻辑，可能行为不一致）

关键问题是：skill 是调用 CLI 还是自己实现的？ 如果 skill 底层调用 openmnemo CLI，那升级 CLI 后 skill 自动受益，不冲突。如果是独立实现，就需要保持两边同步。
