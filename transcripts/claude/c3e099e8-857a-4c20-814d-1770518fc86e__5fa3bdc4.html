---

什么是 npm token

npm token 是 npm（Node.js 包管理平台）颁发的访问凭证，用于在命令行或 CI/CD 流水线中以你的账户身份执行操作，最常见的用途是发布包（npm publish）。

它的工作方式：
- 你登录 npm 网站，在账户设置里生成一个 token（一串随机字符串）
- 把这个 token 配置在 CI/CD（比如 GitHub Actions secrets 里的 NPM_TOKEN）
- 流水线运行 npm publish 时，用这个 token 代替密码进行身份验证

为什么之前要求轮换？

因为 token 曾经在 session 对话记录里出现过（比如命令输出或配置内容被记录进了对话文件里）。一旦 token 泄露，任何拿到它的人都可以：
- 以你的名义发布新版本的包
- 覆盖已有版本（如果是 automation token）
- 污染供应链

轮换的操作：
1. 去 [npmjs.com](https://www.npmjs.com) → Account Settings → Access Tokens → 删除旧 token，生成新 token
2. 在 GitHub 仓库 Settings → Secrets → 把 NPM_TOKEN 的值替换为新 token

如果你已经做了，在 todo 里把那行勾掉就好。

EvoScientist 是一个自主科研 AI 系统（1.1k stars，非常新），核心定位是"自进化 AI 科学家"：给定一个研究课题，6 个 sub-agent（plan/research/code/debug/analyze/write）协作完成从构思到论文的完整科研流程。

---

## 记忆结构分析

关键发现：EvoScientist 的记忆不是向量数据库，而是 Markdown 文件 + LLM 语义匹配。

/memory/
  ideation-memory.md       ← 研究方向记忆（可行 + 失败方向）
  experiment-memory.md     ← 实验策略记忆（数据处理/训练/架构/调试）
  evolution-reports/
    cycle_N_type.md        ← 每轮进化报告

两类记忆：

| 记忆类型 | 存储内容 | 触发更新时机 |
|---------|---------|------------|
| M_I 构思记忆 | 研究方向、成功/失败分类、证据 | idea-tournament 完成后 |
| M_E 实验记忆 | 数据处理策略、训练技巧、调试方法 | 实验完成/失败后 |

"语义检索"实现方式：不是真正的 embedding 向量，而是让 LLM 读每条记录的 Summary 和 Retrieval Tags，用自然语言判断相似度，top-k=2 取最相关的。

---

## 对 OpenMnemo 的启示

EvoScientist 的记忆体系太轻量，不适合直接套用——它是为科研场景设计的，没有跨 session 海量对话检索的需求。

你说"单纯 SQLite 不够，要做向量化"，这个判断是对的。几个本地可自托管的向量方案供参考：

| 方案 | 特点 | 适合 OpenMnemo 的理由 |
|------|------|---------------------|
| sqlite-vec | SQLite 扩展，直接加列存向量 | 保持现有 SQLite 架构，最小改动 |
| LanceDB | 嵌入式，无服务器，支持向量 + 全文混合检索 | 无需额外进程，数据文件可 git 管理 |
| Qdrant | 功能最完整，需独立服务 | 适合团队/SaaS 版，个人版偏重 |

推荐路径：Phase 0 先用 sqlite-vec 扩展现有 db.ts（保持数据层统一），Phase 1 推理层上线时再评估是否引入 LanceDB 做混合检索。
