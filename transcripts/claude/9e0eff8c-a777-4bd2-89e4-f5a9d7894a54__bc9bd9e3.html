Codex Parser Details:
- Reads JSONL records
- session_meta records populate session_id, title, started_at, cwd, branch
- response_item and event_msg records with message payload type extract text
- function_call, custom_tool_call, function_call_output, custom_tool_call_output create tool events
- Calls deduplicate_messages() and deduplicate_tool_events() before returning

Claude Parser Details:
- Reads JSONL records with fields: type (user/assistant), message.role, message.content, timestamp, sessionId, cwd, gitBranch
- Handles content as string or array of blocks
- Text blocks extracted from type: "text" blocks
- Tool use/result blocks create tool events
- Deduplication applied

Gemini Parser Details:
- Reads JSON or JSONL (auto-detected by suffix)
- Recursive visit() function traverses entire structure
- Looks for metadata: sessionId, chatId, cwd, projectRoot, branch, timestamp
- Detects message roles: user, assistant, model (model → assistant)
- Detects tool calls via: toolUse.name, toolName, functionName, tool
- Deduplication applied

Edge Cases:
- Empty content fields yield no messages
- Tool events always created even if timestamp is None
- Gemini handles deeply nested structures via recursive traversal
- Role normalization: Claude/Codex use explicit roles; Gemini standardizes model → assistant

## Key Implementation Details for TypeScript Port

1. Timestamp Handling: All timestamps normalized to ISO 8601 with UTC Z suffix. fromisoformat() compatibility requires converting Z → +00:00.
2. Deduplication: Uses 16-char SHA256 hex prefix as content signature; keyed on (timestamp, role, hash) for messages and (timestamp, hash) for tool events.
3. Path Partitioning: Year/month derived from started_at using format %Y and %m.
4. SQLite Upsert: Primary key is 4-tuple; all fields updated on conflict.
5. Gemini Format: Recursive tree traversal; metadata and messages scattered throughout structure.
6. Claude Format: Array of blocks with role and content; tool use/result are block types.
7. Codex Format: JSONL with explicit record types (session_meta, response_item, event_msg) and payload sub-structure.
