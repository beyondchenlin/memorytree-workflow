## 1. GITEA-RELATED IDEAS (Major Focus)

### Core Vision: Gitea as Universal Office Collaboration Platform

The most significant concept emerging from these documents is transforming Gitea (a lightweight Git platform) into a non-technical user-friendly office collaboration platform. Key ideas:

#### A. Gitea Functionality Mapping to Office Needs

| Office Need | Traditional Tool | Gitea Feature | User View |
|---|---|---|---|
| Project Space | Google Drive | Repository | "My Project" |
| Task Management | Trello | Issues + Projects | "Task Board" |
| Knowledge Base | Notion | Wiki | "Knowledge Center" |
| Document Collaboration | Google Docs | Markdown + PR | "Document Editor" |
| Approval Workflow | Approval Systems | Pull Request | "Approval Request" |
| File Storage | Dropbox | Repository Files + Git LFS | "File Management" |
| Team Discussion | Slack | Issue + Labels | "Team Chat" |
| Version Control | None | Git | "History & Versions" |
| Permissions | Scattered | Teams + Permissions | "Access Control" |

#### B. Architecture Design: "Everything is a Repository"

Every feature, plugin, workflow, and project is stored as an independent Git repository:
- Price monitoring plugin = repository with docker-compose, config, scripts
- Data analysis skill = repository with code, examples, tests
- Personal workspace = dedicated repository for daily memories in Markdown format
- Team project = repository with branches for different people's work

Key Advantages:
- Complete version control (who changed what, when, why)
- Easy one-click deployment via Docker
- Data completely isolated per user/project
- Full Git merge/diff/rollback capabilities
- Team collaboration without conflicts

#### C. Gitea + OpenClaw (AI Brain) Integration

OpenClaw's Role: Translate natural language to Gitea operations

User: "Help me create a competitor analysis project"
↓
OpenClaw (AI):
  1. Create repository: project-competitor-analysis
  2. Initialize folder structure
  3. Clone relevant plugins (price-monitor, news-aggregator, data-analyzer)
  4. Create Issues for setup tasks
  5. Notify team members
  
User sees: Clean interface with project ready to use
(No Git, branches, commits, or technical terms visible)

#### D. PR (Pull Request) + Branch System for Non-Technical Collaboration

Transform Git's powerful collaboration model into office scenarios:

Scenario 1: Document Review
- Employee creates "draft" version in personal workspace
- Submits for approval (= Pull Request)
- Manager reviews with inline comments, version comparison
- Once approved, changes merge to official version
- Complete audit trail: who approved, when, what changed

Scenario 2: Task Completion
- Task assigned to employee
- Employee completes work in personal branch
- Submits for verification (= PR)
- Manager reviews attached deliverables
- If approved, automatically marks task complete and archives result

Scenario 3: Personal Memory Archival
- Daily OpenClaw conversations auto-save to "draft" branch
- OpenClaw summarizes valuable items
- Creates PR to "official memory" branch
- User confirms → merges permanently into personal archive

#### E. Server Requirements

Excellent news: Gitea is extremely lightweight
- Minimum: 256MB RAM, can run on Raspberry Pi
- MVP: 2 cores, 4GB RAM, 50GB SSD = all you need for 10-20 person team
- 100-300MB base memory usage (vs GitLab's 4GB+)
- Private deployment possible (data under your control)

### Tool Navigation Website System

What: An AI tool discovery and comparison website that automatically collects, verifies, and displays information about software tools

Full Pipeline:
1. Discovery Layer: Automatically crawl 20-50 tool directory websites
2. Collection: Extract tool names, URLs, pricing, features from official pages
3. Cleaning: Deduplicate, normalize, resolve conflicts
4. Verification: Two-stage validation with evidence chains (source URL + timestamp + screenshot)

6. Presentation: Web UI with filtering, comparison, sorting
7. Iteration: Continuous re-crawling every 3-30 days

MVP Requirements:
- 1000+ tools with 70%+ key field coverage
- Evidence chain for pricing/free tier/card requirement
- Table view with filtering, sorting, comparison
- All in 4 weeks of implementation

Tech Stack:
- crawler/src/collect.mjs — main scraper
- crawler/src/clean.mjs — deduplication
- Pinchtab (JavaScript rendering for dynamic sites)
- Crawlab (task scheduling)
- Scrapy (optional framework)

Three Execution Phases:
1. Phase A: Single site deep sampling (200 records per site)
2. Phase B: 3-site parallel collection with quality gates
3. Phase C: Full loop (180-min intervals, continuous)

## 5. Key Technical Insights

### Why Gitea Over Alternatives?

vs. Spring若依 Framework:
- 若依 = bare skeleton, need to build everything from zero
- Gitea = complete product, just needs UI wrapper
- Gitea saves 6+ months of development for core features

vs. Notion/Flybook:
- Notion: Self-hosted? No. Customizable? No. Version control? Basic.
- Gitea: All three yes. Data is yours. Can modify everything.

vs. Open Source:
- OpenClaw supplier risk lowered (made optional in Phase 2)
- Gitea is MIT licensed, fully commercial-friendly
- No vendor lock-in

### Data Persistence Model

All information stored as Markdown in Git repositories:
- Conversations → Markdown files in personal repo
- Tasks → Gitea Issues (stored in Git)
- Documents → Wiki or README files
- Approvals → Pull Request history
- Everything: Permanent, versioned, auditable

OpenClaw's Daily Routine:
1. Collect all user conversations
2. Store in temporary "draft" branch
3. Summarize valuable content
4. Create PR to "official memory"
5. Auto-clean temporary files
