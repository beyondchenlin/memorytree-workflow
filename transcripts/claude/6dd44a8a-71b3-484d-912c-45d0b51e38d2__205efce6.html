### 7. Architecture Notes

Immutability Principle:
- Config updates return new objects: registerProject(cfg, path) → new Config
- Alert operations create new arrays: writeAlert() reads, modifies copy, writes
- Failure counts: same pattern

Stateless & Idempotent:
- Heartbeat: single-shot execution, no state between runs
- Lock prevents concurrent execution
- Manifest deduplication by SHA256 prevents re-imports

Platform Support:
- Windows: tasklist for PID checking, handles backslashes in paths
- Unix: kill -0 for PID checking
- Path normalization: toPosixPath converts backslashes to forward slashes

Security:
- Sensitive pattern detection before import
- Alerts on matches (no upload)
- File hashing prevents tampering detection
- Input validation throughout

### 8. Key Implementation Details

Timestamp Handling:
- Unix seconds → ISO 8601 conversion
- Fallback to file mtime if metadata missing
- Earliest timestamp detection for session start
- Partition by YYYY/MM for directory structure

Text Extraction:
- Multi-format support: string, nested objects, arrays
- Deduplication by timestamp + content hash
- Paragraph joining with double newlines
- Summarization with truncation (default 180 chars)

Git Integration:
- Works with any git repo with Memory/ subdirectory
- Commits only if changes exist in Memory/
- Automatic retry on push failure
- Respects auto_push config flag
