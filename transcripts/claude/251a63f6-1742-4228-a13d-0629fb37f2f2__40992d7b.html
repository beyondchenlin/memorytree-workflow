#### 4.1 build.ts:277-280 | MEDIUM | 清洁路径可能为 null 的潜在问题

问题描述：

const cleanPath = m.repo_clean_path
  ? join(root, m.repo_clean_path)
  : m.global_clean_path

缺陷：
- 当 m.repo_clean_path 为空但 m.global_clean_path 存在时，直接赋值
- 但 m.global_clean_path 可能是绝对路径或相对路径，与 repo_clean_path 的处理不一致

  cleanPath = join(root, m.repo_clean_path)
} else if (m.global_clean_path) {
  cleanPath = m.global_clean_path.startsWith('/') ? m.global_clean_path : join(root, m.global_clean_path)
}

#### 5.1 github-pages.ts:88-96 | MEDIUM | makeRelative() 函数不够健壮

问题描述：

function makeRelative(base: string, target: string): string {
  const b = base.replace(/\\/g, '/').replace(/\/$/, '')
  const t = target.replace(/\\/g, '/').replace(/\/$/, '')
  if (t.startsWith(b + '/')) {
    return t.slice(b.length + 1)
  }
  return t
}

缺陷：
- 这是一个简化的相对路径计算，如果 base 和 target 没有公共前缀会返回完整的 target

- 虽然在实际场景中 output 一定在 root 内，但这是脆弱的假设

建议修复：

import { relative } from 'node:path'
// 直接使用 relative() 函数
