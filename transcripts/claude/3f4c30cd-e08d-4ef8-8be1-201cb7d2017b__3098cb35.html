HEARTBEAT

Entry Point: main()

1. Lock Acquisition – Prevents concurrent heartbeat runs
- Uses PID-based lock file at ~/.memorytree/heartbeat.lock
- Checks if stored PID is alive; reclaims stale locks
- Returns 0 if already running (not an error)

2. Load Configuration – From ~/.memorytree/config.toml
- heartbeat_interval (e.g., "5m", default)
- projects – Array of {path, name}
- auto_push – Whether to push transcripts to Git
- log_level – debug\|info\|warn\|error

3. Per-Project Processing – processProject()
- Discover source files from ~/.codex, ~/.claude, ~/.gemini
- Parse each source into ParsedTranscript
- Skip if no content (no messages/tool_events)
- Key: Check if transcript matches repo – via transcriptMatchesRepo()
- Compares cwd in transcript with repo root
- Falls back to slug matching if cwd doesn't match
- Sensitive pattern scanning – Regex patterns for API keys, passwords, tokens
- Import transcript – Calls importTranscript() with:
- parsed transcript
- projectPath (repo root)
- globalRoot (default: ~/.memorytree/transcripts)
- projectSlug (derived from project name)
- rawUploadPermission (not-set\|approved\|denied)
- mirrorToRepo=true – Also store in Memory/06_transcripts/

4. Git Operations – After import
- Check for changes in Memory/ directory
- Commit with message: memorytree(transcripts): import N transcript(s)
- If auto_push enabled: push to remote (with retry)
- Write alerts on failure (threshold-based)

5. Lock Release – Always called in finally block

TRANSCRIPT

Client Inference: inferClient(client, sourcePath)

Auto-detection based on path patterns:
- /.codex/ or filename rollout-* → codex
- /.claude/ or /projects/ → claude
- /.gemini/ or checkpoint → gemini

Three Client-Specific Parsers:

Codex Parser (parseCodexTranscript)
- Format: JSONL records with type field
- Types: session_meta, response_item, event_msg
- Messages: From type=message records (canonical), fallback to type=user_message/agent_message
- Tool events: function_call, custom_tool_call, *_output
- Deduplication: Uses streaming events only if no canonical records (avoids double-recording)
- System injection filter: Skips first message if contains <INSTRUCTIONS>, <environment_context>, etc.

Claude Parser (parseClaudeTranscript)
- Format: JSONL with type field (user or assistant)
- Message content: String or block array with {type, text} objects
- Block types: input_text, output_text, text (text), tool_use, tool_result (tools)
- Skip blocks: thinking, reasoning
- Metadata: sessionId, cwd, gitBranch from any record

Gemini Parser (parseGeminiTranscript)
- Format: JSONL or JSON
- Recursive tree traversal to find:
- Metadata: sessionId, chatId, cwd, branch, timestamp
- Messages: role in (user, model, assistant); extract text from parts, content, text, message
- Tool events: toolName, functionName, tool
- Flexible structure to handle nested conversations

Common Utilities (src/transcript/common.ts):

- Slugify: Removes non-ASCII, converts to lowercase, replaces spaces/special chars with hyphens
- SHA256: File hash (raw), content hash (16-char prefix for dedup)
- Timestamp normalization: Unix seconds → ISO 8601, handle timezone offsets
- Timestamp partition: Extract year/month for directory structure
- Deduplication: Track signature (timestamp\0role\0contentHash) to skip duplicates
- Text extraction: From nested blocks, handles Gemini multi-part messages
- YAML escaping: For frontmatter values

LOCKING

PID-Based Lock File: ~/.memorytree/heartbeat.lock

Function: acquireLock()

1. Check if lock file exists
2. If exists, read PID and check if process is alive:
- Alive → Return false (another instance running)
- Dead → Remove stale lock, continue
3. Use O_CREAT | O_EXCL for atomic creation (prevents TOCTOU race)
4. Write current process PID to file
5. Return true on success

Cross-Platform Process Checking:

- Unix: process.kill(pid, 0) – Signal 0 tests existence
- Windows: tasklist /FI "PID eq {pid}" – Search output for PID word boundary

Functions:

- acquireLock() – Acquire or reclaim stale lock
- releaseLock() – Delete lock file
- readLockPid() – Inspect current lock holder
- isProcessAlive(pid) – Check if process exists

### SUMMARY

MemoryTree Workflow is a comprehensive transcript management system for AI coding assistants. It:

1. Discovers transcripts from ~/.codex, ~/.claude, ~/.gemini
2. Parses client-specific formats into normalized ParsedTranscript objects
3. Imports transcripts into both repo-local (Memory/06_transcripts/) and global (~/.memorytree/transcripts/) storage
4. Indexes entries in an SQLite database for search
5. Runs continuously via heartbeat daemon (cron/launchd/schtasks) to auto-sync new transcripts
6. Commits & pushes transcript archives to Git when changes are detected
7. Manages configuration via ~/.memorytree/config.toml for project registration and heartbeat settings

All components are type-safe (strict TypeScript), well-tested, and cross-platform (Windows/macOS/Linux).
