### 4. Python Scripts (All Implemented)

#### init-memorytree.py (95 lines)
- Creates fresh MemoryTree workspace in any repo
- Arguments: --root, --project-name, --goal-summary, --locale, --date, --time, --force
- Refuses if AGENTS.md or external policy sources exist (directs to upgrade-memorytree.py)
- Uses _scaffold_utils to create directory structure and populate template files

#### upgrade-memorytree.py (306 lines)
- Safely adds missing MemoryTree pieces to partial repos
- Preserves existing AGENTS.md by default
- Outputs JSON or text format with state_before/after, created_files, preserved_files
- Includes 9 sophisticated validation functions to check if AGENTS.md covers required concepts:
- has_read_order_signal()
- has_memory_layout_signal()
- has_active_todo_rule()
- has_append_only_chat_log_rule()
- has_repo_safety_rule()
- has_commit_title_rule()
- has_dedicated_memorytree_pr_flow()
- has_memorytree_only_scope_rule()
- has_auto_merge_rule()

#### import-transcripts.py (121 lines)
- Imports one transcript from a known source file
- Arguments: --root, --source, --client, --project-name, --global-root, --raw-upload-permission, --format
- Determines if transcript belongs to current repo via cwd/source path matching
- Calls import_transcript() from _transcript_utils
- Outputs manifest with all metadata, paths, counts

#### discover-transcripts.py (197 lines)
- Batch discovers transcripts from local client stores (~/.claude/, ~/.codex/, ~/.gemini/)
- Arguments: --root, --client, --scope (current-project|all-projects), --limit, --format
- Mirrors current-project transcripts to both repo and global archive
- Backfills other projects to global archive only
- Returns import results, skipped items with reasons

#### detect-memorytree-locale.py (42 lines)
- Detects effective locale for a repo without scaffolding files
- Arguments: --root, --locale, --format (text|json)
- Returns requested/repo_locale/system_locale/effective_locale
- Pure detection, no file creation

#### _locale_utils.py (124 lines)
- normalize_locale() - alias mapping (en-us→en, zh-tw→zh-cn, etc.)
- detect_repo_locale() - inspects AGENTS.md, README.md, top-level files, docs/
- detect_system_locale() - checks locale.getlocale(), LC_ALL, LANG env vars
- CJK character detection (3400-9FFF Unicode range)
- Latin character detection
- Scoring system: if zh_score ≥ en_score × 1.25 → zh-cn, vice versa

#### _scaffold_utils.py (320 lines)
- build_datetime() - parse --date and --time arguments
- resolve_template_dir() - locate locale-specific templates
- find_external_policy_sources() - detect CODEOWNERS, PR templates, commitlint, CONTRIBUTING with policy keywords
- find_latest_goal_file(), find_latest_todo_file(), find_latest_chat_log_file() - version finding logic
- create_memory_dirs() - create Memory/01_goals through Memory/05_archive
- scaffold_content_files() - populate goal, todo, chat-log with template substitutions
- write_template() - replace {{PLACEHOLDER}} patterns in files

#### _transcript_utils.py (927 lines) - Most Complex
Comprehensive transcript handling:

Data Classes:
- TranscriptMessage(role, text, timestamp)
- TranscriptToolEvent(summary, timestamp)
- ParsedTranscript(client, session_id, title, started_at, cwd, branch, messages, tool_events, source_path)

Parsing Functions (supports 3 clients):
- parse_codex_transcript() - JSONL format with type/payload/timestamp
- parse_claude_transcript() - JSONL with sessionId, message blocks, tool_use/tool_result
- parse_gemini_transcript() - JSON or JSONL with recursive visitor pattern

Import Logic:
- import_transcript() - master function that:
- Copies raw file to both repo mirror and global archive
- Generates cleaned Markdown deterministically (no model tokens)
- Creates manifest JSON with metadata
- Upserts SQLite search index (WAL mode for concurrency)
- Preserves existing import_timestamp if manifest unchanged

Search Index:
- upsert_search_index() - SQLite with ON CONFLICT DO UPDATE
- Table: transcripts (client, project, session_id, raw_sha256, ...)

Discovery:

- transcript_matches_repo() - matches via cwd or project slug
- infer_project_slug() - extracts from cwd, Claude source path, or Gemini parent

Utilities:
- slugify() - ASCII-only, lowercase, replace non-alphanumeric with -
- sha256_file() - content hash for deduplication
- normalize_timestamp() - ISO 8601 with Z suffix
- timestamp_partition() - YYYY/MM for directory structure
- yaml_escape() - escape quotes/newlines for YAML frontmatter
- extract_text_blocks() - recursively pull text from nested content
- extract_gemini_text() - Gemini-specific text extraction

### 7. Reference Documentation

project-detection.md:
- Three states: not-installed, partial, installed
- Partial = AGENTS.md exists OR Memory/ exists but incomplete
- Installed = all of 01_goals, 02_todos, 03_chat_logs exist + AGENTS.md ready

memory-layout.md:
- Folder structure: Memory/{01_goals,02_todos,03_chat_logs,04_knowledge,05_archive,06_transcripts/}
- Naming: goal_vNNN_YYYYMMDD.md, todo_vNNN_SSS_YYYYMMDD.md, YYYY-MM-DD_HH-MM.md
- 06_transcripts subdivided: clean/, manifests/, raw/

git-policy.md:
- Default: stage only MemoryTree files, use dedicated branch, MemoryTree-scoped commit
- Safeguards: never mix product code, obey repo's branch/PR/CI rules
- Raw transcripts excluded from staging until user explicitly approves per-repo
- Auto-merge only when repo still enforces required checks

locale-selection.md:
- Priority: explicit --locale > repo content > system locale > en
- Repo inspection: AGENTS.md, README.md, README.zh-CN.md, top-level files, docs/
- CJK threshold: 8+ CJK chars and 4× Latin ratio triggers zh-cn

response-language.md:
- Priority: user request > user's message language > repo locale > active template locale > en
- Never rewrite existing files just to translate them
- Keep file language stable, mirror user's discussion language in replies

update-rules.md:
- New goal only on scope change (not routine progress)
- Update todo on progress/milestone/next-step change
- Chat logs append-only
- Move stale context to knowledge/archive
- Move domain facts/decisions to knowledge

transcript-archive.md:
- Three layouts: repo mirror, global archive, and per-project storage
- Rules: preserve raw unchanged, generate cleaned with code, keep repo mirror current-project-only
- Global archive: ~/.memorytree/transcripts/{raw,clean,index}/
- One-time approval per repo for raw transcript uploads

upgrade-path.md:
- Merge-safe: create missing dirs/files, preserve existing AGENTS.md
- Conservative detection: only "ready" if AGENTS.md covers 8 concepts
- Output: state_before/after, created_files, preserved_files, agents_merge_required

execution-environment.md:
- Preferred runners: uv run python > system python > manual scaffolding from templates
- Never auto-create .venv or add tooling files without user consent
- For fresh init: fall back to templates if no runner available
- For upgrade/detection: manual review if no runner available

### 9. Key Design Decisions

1. Deterministic over AI-Generated: Transcript cleaning uses code, not model tokens. Raw transcripts preserved as source of truth.

2. Non-Destructive Upgrade: Existing AGENTS.md and policy files are preserved and validated, not overwritten.

3. Concurrent-Safe Global Archive: Uses SQLite WAL mode for multi-project updates without corruption.

4. Repo-Scoped Mirrors: Current project's transcripts mirrored into repo; other projects archived globally only.

5. Client Auto-Detection: Detects Codex/Claude/Gemini from path patterns without user specification.

6. Locale Auto-Selection: Detects from repo content (CJK character scoring) before system locale.

7. Zero Dependencies: All scripts use only Python standard library (json, sqlite3, pathlib, re, datetime, etc.).

8. Safe Git Defaults: Isolated PR flow, never bypasses repo rules, raw transcripts unapproved by default.
