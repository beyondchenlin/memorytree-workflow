常见有 3 种分法。

1. 按域名分
最标准。

例如：

- memory.xxx.com -> MemoryTree
- api.xxx.com -> 你的接口服务
- admin.xxx.com -> 后台

外面看都是 80/443 进来，但 Caddy 会按域名分流。

2. 按路径分
适合同一个域名下挂多个服务。

例如：

- example.com/memory -> MemoryTree
- example.com/api -> API
- example.com/admin -> 后台

这时候还是一个入口，只是按 URL 路径分。

3. 按端口分
最简单，但最土。

例如：

- localhost:10010 -> MemoryTree
- localhost:10011 -> 另一个站
- localhost:10012 -> 再一个站

这个也能用，但端口会越来越多，不够优雅。

你当前这个场景，最重要的原则是：

- 对外端口最好由 Caddy 占
- 具体项目服务不要都直接抢外部端口
- 项目自己的服务如果需要跑，就跑内部端口，比如 3000、4000
- Caddy 再转发过去

比如：

- Caddy 占 10010
- A 服务实际跑 3000
- B 服务实际跑 4000
- Caddy 负责把不同请求转给 3000 或 4000

- Caddy 不转发
- 直接把不同目录当成不同站点来托管
