- The README explicitly says the project supports Codex.

  - `~/.claude/skills/memorytree-workflow`
  - Windows variant under `$USERPROFILE/.claude/skills/memorytree-workflow`
- `package.json` requires Node `>=20` and the CLI is built from TypeScript into `dist/cli.js`
- `SKILL.md` is located at the repository root, which means the repo root is the skill directory

If a user in Codex follows the current README literally:

1. They clone the repo into `~/.claude/skills/memorytree-workflow`
2. They run `npm install && npm run build`
3. The build succeeds
4. Codex still does not pick up the skill, because it is installed under the Claude Code skill directory rather than the Codex skill directory

5. Run `npm install`
6. Run `npm run build`
7. Restart Codex

- The fact that this repository root is the skill directory
